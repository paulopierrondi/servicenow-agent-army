# Gallery — ServiceNow Agent Army outputs

Public reference library of artifacts produced by the agent army. Marketing
proof and a starter library for less experienced ServiceNow builders.

Every entry is a sanitized output captured from a real council deliberation.
Customer names are replaced with industry archetypes ("FSI Brazil tier-1
bank", "Brazilian retail chain"). Numbers in `outcomeMetric` are defensible
against the `docs/best-practices/` material. Where a number is a working
hypothesis, the example marks it as such.

## What is in the gallery

| Slot | Purpose |
| --- | --- |
| `case.md` | Original problem, persona, context, council deliberation summary, outcome target |
| `agent-spec.json` | AI agent spec following `templates/agent-spec.template.json` |
| `workflow.json` | Agentic workflow following `templates/workflow-spec.template.json` |
| `skill-prompt.md` | Now Assist Skill Kit (NASK) prompt ready to paste |
| `now-assist-hook.md` | Surface mapping, configuration minima, ROI vs DIY, when not to use Now Assist |

## Schema for `catalog/gallery.json`

```json
{
  "id": "incident-triage-fsi",
  "title": "Incident triage with Now Assist for ITSM (FSI Brazil)",
  "domain": "ITSM",
  "personaUsed": "service desk analyst",
  "agentsParticipated": ["business-analyst-agent", "cta-agent"],
  "nowAssistSurface": "Now Assist for ITSM",
  "summary": "One-paragraph plain-English description.",
  "outcomeMetric": "Defensible quantitative target with timeframe.",
  "guardrailHighlights": ["Bullet list of mandatory guardrails."],
  "path": "gallery/01-incident-triage-fsi/",
  "submittedBy": "ServiceNow Agent Army team",
  "submittedAt": "2026-04-26"
}
```

`agentsParticipated` IDs must exist in `catalog/agents.json`.

## How to contribute

1. Fork the repo, branch from `main`.
2. Add a numbered directory under `gallery/` with the five files above.
3. Add an entry in `catalog/gallery.json`. Reuse existing agent IDs.
4. Sanitize: no customer names, no real CPF/CNPJ/account numbers, no internal
   project codes. Replace with archetypes.
5. Cite the relevant `docs/best-practices/*.md` section in
   `now-assist-hook.md` when you claim ROI or compare against DIY.
6. Open a PR with the title `gallery: <slug> — <one-line outcome>`.

## Disclosure

Outputs were generated by the council of agents in this repo. Each was
sanitized by a human reviewer before publication. Customer-identifying
material was removed; structural details, architectures, and metric
hypotheses are preserved. Where a number could not be backed by published
ServiceNow material or in-repo documentation, the example is marked as a
working hypothesis rather than a guaranteed outcome.

This gallery is opinionated, not endorsed by ServiceNow. The Constructive
Challenger voice is intentional: the goal is to teach pattern recognition,
not to sell.
