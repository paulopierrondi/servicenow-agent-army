# Launch Copy

## One-Liner

ServiceNow Agent Army helps teams create AI agents and agentic workflows with prompts, specialists, guardrails, and SDK-ready scaffolds.

## Short Pitch

Most teams want ServiceNow AI agents, but the first step is messy: unclear requirements, vague prompts, risky automations, and too much platform context for beginners.

ServiceNow Agent Army turns that chaos into a repeatable builder flow. A CTA Agent shapes architecture, a Business Analyst Agent captures intent, an Enterprise Architect Agent checks fit, a Workflow Composer drafts AI Agent Studio steps, a Guardrails Reviewer catches risk, and a Token Saver Specialist makes the prompt lean enough to actually use.

The result: better agent specs, safer workflows, and faster demos.

## LinkedIn Post

I am building ServiceNow Agent Army: a public starter kit for teams that want to create ServiceNow AI agents and agentic workflows with Codex and Claude.

The idea is simple: not everyone on a team is a CTA, BA, EA, prompt engineer, and ServiceNow SDK developer at the same time.

So the repo gives you a small army of specialist agents:

- CTA Agent for architecture decisions
- Business Analyst Agent for requirements
- Enterprise Architect Agent for governance and data fit
- Workflow Composer for AI Agent Studio flow design
- ServiceNow SDK Builder for source-driven artifacts
- Token Saver Specialist for lean prompts
- Guardrails Reviewer for safety
- Demo Storyteller for adoption and enablement

Goal: let less experienced builders create better ServiceNow agent specs with prompts, while keeping production changes behind review, dry run, and approval.

Public repo: https://github.com/paulopierrondi/servicenow-agent-army
